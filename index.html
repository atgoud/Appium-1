Welcome to MTAF Pages.

Mobile Test Automation Framework (MTAF) is a test automation framework which provides seamless testing of a wide range of web-browsers, mobile browsers and native mobile applications such as iOS and Android application:

Example, Create a TimeoutsConfig instance. You can also just use TimeoutsConfig.defaultTimeoutsConfig().

TimeoutsConfig timeouts = TimeoutsConfig.builder()
    .clickTimeoutSeconds(2)
    .webElementPresenceTimeoutSeconds(5) 
    .pageLoadTimoutSeconds(10) 
    .implicitWaitTimeoutMillis(2000)
    .build();

Example, JavaScripts in Selenium

Browser browser = LocalBrowserBuilder.getChromeBuilder("http://my.webapp.com/webapp")  // Base URL for testing. 
    .withTimeoutsConfig(timeouts)             // TimeoutsConfig created above.
    .withBrowserLocale(Locale.US.toString())  // Browser locale
    .withStartWindowWidth(1280)               // Starting width for the browser window in pixels
    .withStartWindowHeight(1024)              // Starting height for the browser window in pixels
    .withBrowserLogLevel(Level.INFO)          // Logging Level for the WebDriver's logs
    .withBrowserLogFile("chromedriver.log")   // Path to logfile, only supported for Chrome and IE. 
    .build();

// Load a web page
TopLevelPage googleHomePage = browser.openPageByUrl("http://google.com");

Example, Sample code creating a Mobile instance (Android) in PageObject/PageFactory

TimeoutsConfig timeouts = getTimeoutsConfig();
MobileBrowser mobile = createAndroidTest(timeouts);
mobile.getActions().loadTopLevelPage(LoginPage.class).login();
mobile.quit();

private MobileBrowser createAndroidTest(TimeoutsConfig timeouts) throws WebDriverException {
       return MobileBrowserBuilder.getAndroidBuilder(android.getAppUrl())
             .withTimeoutsConfig(timeouts)
             .withPlatformVersion(android.getPlatformVersion())
             .withDeviceName(android.getDeviceName())
             .withApp(android.getAppPath())
             .withAppPackage(android.getAppPackage())
             .withAppActivity(android.getAppAcitivity())
             .build();
private TimeoutsConfig getTimeoutsConfig() {
     return TimeoutsConfig.builder()
             .pageLoadTimoutSeconds(10)
             .implicitWaitTimeoutMillis(20000)
             .build();
}

SeleniumActions

  • SeleniumActions are for interacting with the DOM and javascript of a page.
  • Obtain a SeleniumActions from a Browser instance with browser.getActions()
  • Provides methods for waiting until a WebElement is present or visible.
  • Provides methods to refresh the page until a WebElement is present or visible.
  • Provides methods for finding a WebElement containing specific text.
  • Provides methods for interacting with Tiny MCE text editors.
  • Provides methods to load Pages.
  • Provides much more functionality.
  • Pages

Pages provide an abstraction for modeling the pages for your webapp. Pages should extend BaseTopLevelPage or BaseSubPage. Uses the Selenium @FindBy annotation to instantiate member variables that are WebElements. Use the annotation @SubPageField to indicate a member variable that is a SubPage and should be instantiated on page load. Model the actions that you can perform on your web pages in your Page classes. Then, test code is incredibly simple. It just delegates to Page classes and performs high-level actions.
